Stage 2 — Detector Algorithms (detectors/)
BaseDetector  (abstract, detectors/base.py)
  • absupdate(value: float) → Tuple[bool, float]
  • absreset() → None
Returns (drift_detected, drift_score)
All four concrete detectors inherit this interface.
🌊
ADWIN
Adaptive Windowing
Maintains a sliding window. For every split point, computes left/right means and compares to a Hoeffding-bound threshold ε. Drift = |m₀ − m₁| ≥ ε. Discards left portion on drift.
Paramsdelta=0.002, max_window=10k
MemoryO(max_window)
SpeedO(n) per update
DetectsGradual mean shift
WarmupGradual
Statedeque[float], total: float
Memory usageHigh
📈
PageHinkley
Cumulative Sum Test
Tracks cumulative deviation from running mean minus δ. Drift = U > λ, where U = cumsum − min_cumsum. Exponentially weighted mean via α forgetting factor.
Paramsdelta=0.005, lambda_=50, alpha=1.0
MemoryO(1) — 3 scalars
SpeedO(1) per update
DetectsSudden mean shift
WarmupImmediate
Staten, mean, cumsum, min_cumsum
Memory usageMinimal
📊
KSWIN
KS-Windowed Test
Runs a two-sample KS test between the most recent stat_size values and a reference window. p-value < α → drift. stdlib KS implementation (no scipy).
Paramsalpha=0.005, window=100, stat=30
MemoryO(window_size)
SpeedO(n·log n) per update
DetectsAny distribution change
Warmupwindow_size observations
Statedeque[float], reference list
Memory usageLow
🎯
DDM
Drift Detection Method
Monitors binary error rate. Drift = pₜ + σₜ > p_min + drift_level × σ_min. Tracks minimum baseline (p_min, σ_min) since last reset.
Paramswarn=2.0, drift=3.0, min=30
MemoryO(1) — 5 scalars
SpeedO(1) per update
DetectsClassifier error rate ↑
Warmupmin_num_instances (30)
Staten, p, σ, p_min, σ_min
Memory usageMinimal
Stage 3 — Drift Classifier (classifier.py)
DriftClassifier
Maintains per-feature score history (deque, maxlen=50). Priority: sudden › recurring › incremental › gradual.
  • pubrecord(feature, drift_detected, score) → None
  • pubclassify(drifted_features) → Optional[str]
  • pubreset() → None
  • prv_classify_feature(feature) → str
⚡
Sudden
Last 2 scores jump: prev < 0.2 → curr > 0.6. Abrupt distribution change.
🔁
Recurring
Feature has drifted more than once total (_drift_count > 1). Cyclic or seasonal concept drift.
📉
Incremental
Last 5 scores are monotonically increasing. Slow continuous drift over time.
🌫️
Gradual
Default fallback. Slow change not fitting the above patterns.
Stage 4 — Retraining Window Engine (window_engine.py)
RetrainingWindowEngine
Finds optimal stable data window for model retraining after drift is confirmed.
  • pubfind_window(score_history, drift_index) → Optional[RetrainingWindowResult]
Algorithm
1 Walk backward from drift_index
2 Stable = all features' max score < sensitivity
3 Stop at first unstable observation
4 Trim buffer_pct × n (10%) from each end
5 Return None if < 2 stable samples remain
6 Confidence = fraction of truly stable samples in raw region
Constructor Parameters
sensitivity0.002 — stable threshold per feature
buffer_pct0.10 — 10% trim from each end

sklearn Integration (sklearn_wrapper.py)
DriftDetector  pass-through transformer
Wraps DriftMonitor as a sklearn-compatible TransformerMixin. Data passes through unchanged — drift events are handled purely via callbacks.
  • pubfit(X, y=None) → self  initialise monitor, set reference
  • pubtransform(X, y=None) → X  run monitor, return X unchanged
  • pubget_params(deep=True) → dict
  • pubset_params(**params) → self
  • propmonitor → DriftMonitor
Usage in sklearn Pipeline
from pattern_drift import DriftDetector from sklearn.pipeline import Pipeline pipe = Pipeline([ ("drift", DriftDetector( method="ADWIN", callbacks=[log_callback()] )), ("model", your_estimator), ]) pipe.fit(X_train) pipe.transform(X_stream) # drift detected silently
requires: scikit-learn ≥ 1.1  (optional extra: pip install pattern-drift[sklearn])
